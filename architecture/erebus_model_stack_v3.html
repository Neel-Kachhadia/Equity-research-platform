System Architecture Overview
Design Principle

The EREBUS model stack is designed around three interlocking pillars: Retrieval (find the right evidence), Quantitative Analysis (compute structured signals), and Language Generation (synthesise explainable, source-cited output). Every model in scope serves at least one of these pillars. Models that serve trading, allocation, or price prediction are explicitly out of scope.

Model Group 1
Retrieval & Knowledge Layer

The retrieval layer is the epistemic foundation of EREBUS. It is the mechanism by which the platform grounds every generated insight in verifiable source documents — annual reports, earnings call transcripts, regulatory filings. Without this layer, the LLM operates on parametric memory alone; with it, every claim can be traced to a specific document, page, and section.

Model 01-A · Core
Document Embedding Model
text-embedding-3-large · BGE-M3 · E5-large-v2
Purpose
Converts document chunks (paragraphs, table rows, MD&A sections) into dense vector representations. These vectors capture semantic meaning — enabling retrieval by concept, not keyword. All annual reports and earnings transcripts are pre-chunked and embedded at ingestion time.
Core RAG
Model 01-B · Core
Vector Retrieval Engine
FAISS · Qdrant · OpenSearch kNN
Purpose
Given a user query or analytical question, retrieves the top-k most semantically relevant document chunks. Retrieval is filtered by company, document type, and date range. Results carry source metadata (document, page, section) that is injected into the LLM prompt and preserved in final output citations.
Core RAG
Chunk StrategySemantic chunking with 512-token windows, 64-token overlap. Tables extracted separately and embedded as structured text.
Index ScopeAnnual reports (FY20–FY24), quarterly earnings transcripts, SEBI filings, investor presentations.
Retrieval ModeHybrid: dense vector search + BM25 keyword fallback. Reranker (Cohere Rerank or BGE-Reranker) applied to top-20 candidates before final top-5 selection.
Source CitationEvery retrieved chunk carries: doc_id, page_num, section_title, chunk_text. Citations mandatory in all LLM output.
C
Model Group 2
Large Language Model

The LLM is the synthesis engine. It receives retrieved document evidence, computed quantitative signals, and structured alpha scores — and produces analyst-grade, explainable written output. The LLM does not perform calculations directly; it narrates and reasons over pre-computed outputs from the quantitative layer.

Model 02 · Core
Large Language Model
Amazon Bedrock (Claude) · Anthropic Claude · OpenAI GPT-4o
Purpose
Generates analyst-style written explanations combining quantitative signals with retrieved documentary evidence. Produces structured outputs: risk assessments, guidance credibility analysis, factor decompositions, executive summaries. Operates under strict source-citation requirements — every factual claim must reference a retrieved document chunk.
Core RAG Language
Scope Constraint

The LLM is explicitly constrained from making investment recommendations, price predictions, or portfolio allocation suggestions. Its role is to explain and synthesise — not to advise. System prompts enforce this boundary at inference time with refusal logic for out-of-scope queries.

Input ContextRetrieved chunks (top-5), pre-computed alpha scores, risk score, financial ratios, trend signals, sentiment score — all structured and injected as system context.
Output FormatStructured JSON (for API consumers) + Markdown (for report rendering). Every factual claim tagged with source citation ID.
DeploymentAmazon Bedrock (primary) for AWS-native security posture. OpenAI GPT-4o as fallback. Model abstraction layer allows hot-swapping.
GuardrailsConstitutional AI prompting. Refusal for: price targets, portfolio weights, buy/sell recommendations, insider information queries.

Model Group 3
Financial Ratio Models

The financial ratio layer is the analytical bedrock of the quant stack. These are deterministic, formula-based computations applied to structured balance sheet and income statement data. They are the raw inputs from which all higher-order signals (alphas, risk scores) are derived.

Produce the fundamental financial features that feed the alpha signal engine, risk scorer, and time-series trend models. Base layer for all comparative analysis.

Model Group 4
Time-Series Trend Models
Model 04-A · Core
Linear Regression (Trend Slope)
scikit-learn · OLS · Rolling Windows
Purpose
Fits a linear trend line to 5-year historical series of key metrics (revenue, margins, ROE). The slope coefficient indicates directional momentum — rising, flat, or declining. Enables the platform to distinguish a structurally improving business from one masking deterioration.
Core Quant
Model 04-B · Core
Moving Average
3-Year / 5-Year Trailing Average
Purpose
Smooths annual metric volatility to surface underlying performance trajectory. Moving average deviation (current vs trailing avg) is a key input to the Relative Strength Alpha and the Consistency model. Removes single-year distortions caused by one-off events.
Core Quant

Model Group 5
Volatility Models
Model 05-A · Core
Standard Deviation of Returns
Rolling 3-Year σ on Key Metrics
Purpose
Measures the year-on-year variability of revenue growth, margins, and earnings. High standard deviation signals an unpredictable business. Feeds directly into the Volatility Alpha and Risk Score as a negative quality signal.
Core Risk
Model 05-B · Core
Parkinson Volatility Estimator
High-Low Range Based Estimator
Purpose
A more efficient volatility estimate using the range of a metric across reporting periods. Particularly effective when sample sizes are small (5-year history), as it extracts more information per data point than standard deviation alone. Supplements the σ model for robustness.
Core Risk Quant

Model Group 6
Risk Scoring Model

The risk scoring model transforms multiple quantitative signals into a single, interpretable risk classification. It is rule-based and weighted — intentionally transparent, so that risk assessments can be explained to institutional clients without reference to a "black box."

Model 06 · Core
Weighted Risk Score
Rule-Based · Interpretable · Weighted Composite
Input Signals
Debt-to-Equity level and trend · Net Profit Margin trend · Revenue volatility (σ) · Interest Coverage Ratio · FCF Conversion stability · Sentiment negativity score
Output
Risk classification: LOW / MEDIUM / HIGH, with numeric score 0–100 and factor-level breakdown explaining each contribution to final score.
Core Risk
Scoring MethodWeighted sum of normalised factor scores. Weights calibrated against sector benchmarks. Debt signals carry highest weight (30%), margin trend second (25%).
ThresholdsLOW: 0–33 · MEDIUM: 34–66 · HIGH: 67–100. Thresholds are configurable per sector (e.g., banking D/E norms differ from FMCG).
ExplainabilityWaterfall chart showing each factor's contribution to the final score. No black-box outputs. Every score is decomposable and auditable.

Model Group 7 · Core Differentiator
Alpha Signal Models

The alpha signal layer is the primary differentiator of the EREBUS platform. Rather than presenting raw financial ratios to users, the system converts them into named, scored, directional signals — each with a clear economic interpretation. These signals are the language through which EREBUS communicates analytical judgment.

Design Rationale

Alpha signals sit between raw data and narrative. They transform numbers (e.g., "margin declined 2.3pp") into scored, directional assessments (e.g., "Margin Alpha: −14, deteriorating trend over 3 years") that the LLM can reason over and the risk model can incorporate. They are the semantic bridge between the quant layer and the language layer.

Alpha 01
Growth Alpha
Revenue and earnings growth rate vs sector median and own 5-year history. Captures absolute and relative growth quality.
Alpha 02
Margin Alpha
Trend in gross, operating, and net profit margins. Distinguishes expanding vs compressing margin structures.
Alpha 03
Consistency Alpha
Regularity and predictability of growth. Low variance, high hit rate. Rewards steady compounders over erratic performers.
Alpha 04
Risk Alpha
Composite of leverage, coverage, and margin stability signals. Negative alpha for high-debt, low-coverage businesses.
Alpha 05
Volatility Alpha
Inverse of earnings and revenue volatility. Stable businesses score positively; cyclical / erratic ones score negatively.
Alpha 06
Credibility Alpha
Management Credibility Score (MCS) — tracks guidance vs actual delivery over historical periods. High MCS = trustworthy forward-looking statements.
Alpha 07
Sentiment Alpha
FinBERT-derived tone score from earnings call transcripts. Tracks language trajectory: improving / stable / deteriorating confidence signals.
Alpha 08
Relative Strength Alpha
Performance vs sector peers on key metrics. Identifies outperformers and laggards within peer groups. Feeds cross-sectional ranking.
Score RangeEach alpha is normalised to −100 → +100. Positive = bullish signal, negative = bearish. Zero = neutral / insufficient data.
Composite ScoreWeighted combination of all 8 alphas produces a single Composite Alpha Score (CAS). Weights are sector-configurable. Default: Growth (20%), Margin (20%), Risk (15%), Credibility (15%), Consistency (15%), Sentiment (10%), Volatility (5%), Relative Strength (Rel.Str.) (5%).
ExplainabilityEach alpha score accompanied by textual rationale generated by the LLM, grounded in retrieved document evidence where applicable.

Alpha Signal Formulas

Each alpha is computed deterministically from the underlying financial inputs, then normalised to the −100 → +100 range via percentile rank within the active universe. Formulas below show the raw computation before normalisation.

α₁ Growth Alpha
GrowthAlpha = 0.5 × CAGR_rev(5yr) + 0.3 × CAGR_ebit(5yr) + 0.2 × Δrev_vs_sector_median ───────────────────────────────────── CAGR = (V_n / V_0)^(1/n) − 1 Δrev_vs_sector: company CAGR − peer median CAGR
α₂ Margin Alpha
MarginAlpha = 0.4 × Δ_NPM(t, t−3) + 0.35 × Δ_EBIT_margin(t, t−3) + 0.25 × Δ_gross_margin(t, t−3) ───────────────────────────────────── Δ_M = M_t − M_{t-3} (pp change over 3yr) Positive = expanding margin structure
α₃ Consistency Alpha
ConsistencyAlpha = − CV_rev × 0.5 − CV_ebit × 0.3 + HitRate × 0.2 ───────────────────────────────────── CV = σ / |μ| (coefficient of variation) HitRate = years YoY growth > 0 / total years
α₄ Risk Alpha
RiskAlpha = −0.30 × norm(D/E) −0.25 × norm(1 / ICR) −0.25 × norm(σ_margin) + 0.20 × norm(FCF_conv) ───────────────────────────────────── ICR = EBIT / Interest Expense FCF_conv = FCF / Net Income
α₅ Volatility Alpha
VolatilityAlpha = −0.5 × σ_rev_growth(3yr) −0.3 × Parkinson_vol −0.2 × σ_ebit_margin(3yr) ───────────────────────────────────── Parkinson = √(1/4n·ln2 · Σln(H_i/L_i)²) Higher volatility → more negative score
α₆ Credibility Alpha
CredibilityAlpha = 0.5 × GuidanceAcc + 0.3 × SentimentTrend − 0.2 × HedgeLanguageRatio ───────────────────────────────────── GuidanceAcc = hits / total guidance points HedgeRatio = hedge tokens / total tokens
α₇ Sentiment Alpha
SentimentAlpha = 0.5 × FinBERT_pos_ratio(t) + 0.3 × Δ_sentiment(t, t−2) − 0.2 × FinBERT_neg_ratio(t) ───────────────────────────────────── pos_ratio = positive sentences / total Δ_sentiment = current score − 2yr avg score
α₈ Relative Strength Alpha
RelStrengthAlpha = 0.4 × PctRank_rev_growth + 0.35 × PctRank_NPM + 0.25 × PctRank_ROE ───────────────────────────────────── PctRank = percentile within sector universe Scaled: rank_pct × 2 − 1 → [−1, +1]

Composite Alpha Score — CAS Equation

The Composite Alpha Score (CAS) aggregates all eight normalised alphas into a single scalar via a sector-configurable weighted sum. The result is further bounded to the [−100, +100] range and passed to the cross-sectional ranking model and LLM context layer.

CAS = Σ wi × αi where Σ wi = 1 CAS = 0.20·α₁ + 0.20·α₂ + 0.15·α₃ + 0.15·α₄ + 0.15·α₅ + 0.10·α₆ + 0.05·α₇ + 0.05·α₈ αᵢ ∈ [−100, +100] → CAS ∈ [−100, +100] · clamp(CAS, −100, +100)
α₁ Growth Alpha · w=0.20
α₂ Margin Alpha · w=0.20
α₃ Consistency Alpha · w=0.15
α₄ Risk Alpha · w=0.15
α₅ Volatility Alpha · w=0.15
α₆ Credibility Alpha · w=0.10
α₇ Sentiment Alpha · w=0.05
α₈ Rel. Strength Alpha · w=0.05
Sector Weight Overrides

Default weights are calibrated for general industrials and consumer sectors. Banking / NBFC overrides reduce Growth weight to 0.12, elevate Risk Alpha to 0.22 and Credibility Alpha to 0.18 to reflect the capital-intensive nature of financial institutions. Weights are stored in a sector config YAML and applied at query time.

Confidence Score Data Confidence Index — DCI Derived Signal · V1.0

The Data Confidence Index (DCI) quantifies how much trust should be placed in the Composite Alpha Score for a given company. It penalises thin data histories, missing fields, and high cross-alpha disagreement. The LLM receives DCI alongside CAS and is instructed to hedge its language proportionally when DCI is low.

DCI = w_cov × DataCoverage + w_hist × HistoryDepth + w_agree × AlphaAgreement DataCoverage = available fields / required fields ∈ [0, 1] HistoryDepth = min(years available / 5, 1) ∈ [0, 1] AlphaAgreement = 1 − (std_dev of signed αᵢ / 100) ∈ [0, 1] Weights: w_cov = 0.40 · w_hist = 0.35 · w_agree = 0.25 DCI ∈ [0, 1] → displayed as 0–100%
DCI ≥ 0.80
High Confidence
≥5yr history, all fields present, alphas broadly aligned. CAS presented without qualifications.
DCI 0.50–0.79
Moderate Confidence
Partial data or 3–4yr history. LLM flags data gaps. CAS score displayed with caution band.
DCI < 0.50
Low Confidence
Thin data or strong alpha disagreement. CAS suppressed in ranking output. Analyst review recommended.
Divergence Signal Quant–Sentiment Divergence — QSD Derived Signal · V1.0

The Quant–Sentiment Divergence (QSD) flags cases where the quantitative alpha signals (α₁–α₅) point in a materially different direction from the language-derived signals (α₆–α₇). High divergence is a structurally interesting signal: it may indicate a company whose narrative has decoupled from its fundamentals — either a recovery story the numbers haven't yet confirmed, or a deteriorating business whose management is still projecting confidence.

QSD = CAS_quant − CAS_sentiment CAS_quant = normalised(α₁·0.28 + α₂·0.28 + α₃·0.20 + α₄·0.14 + α₅·0.10) CAS_sentiment = normalised(α₆·0.60 + α₇·0.40) QSD ∈ [−200, +200] · |QSD| > 40 triggers divergence flag QSD > 0 → quant stronger than sentiment · QSD < 0 → sentiment stronger than quant
QSD > +40
Narrative Lag
Fundamentals outperform tone. Management may be overly cautious or market narrative has not caught up. Potential underappreciated quality.
|QSD| ≤ 40
Aligned
Quantitative and sentiment signals are broadly consistent. High-conviction CAS output. No divergence adjustment applied.
QSD < −40
Narrative Excess
Tone outperforms fundamentals. Management may be over-projecting confidence. Elevate scrutiny; raise Risk Alpha weight by +5pp in CAS recalculation.
LLM Instruction — Divergence Handling

When |QSD| > 40, the LLM is explicitly instructed to surface the divergence in its analysis section: "Note: A significant gap exists between fundamental signals (CAS_quant: X) and management tone signals (CAS_sentiment: Y). This may indicate [narrative lag / narrative excess]. Analyst review of recent management commentary against reported results is advised."

Model Group 8 & 9
Correlation & Ranking
Model 08 · Core
Pearson Correlation
scipy.stats · Rolling Pairwise Correlation
Purpose
Identifies relationships between companies' financial metrics — e.g., whether two banks' NIM trajectories are correlated, or whether a supplier's revenue growth tracks a customer's capex cycle. Enables comparative analysis and contagion risk identification. Not used for portfolio allocation.
Core Quant
Model 09 · Core
Cross-Sectional Ranking
Percentile Rank · Z-Score Normalisation
Purpose
Ranks companies within a sector on each alpha signal and on composite score. Percentile-normalised so that rankings are comparable across sectors with different absolute metric ranges. Output: ranked league table with factor-level attribution explaining each company's position.
Core Quant
J
Model Group 10
Sentiment Analysis Model
Model 10 · Core NLP
Financial Sentiment (FinBERT)
FinBERT · ProsusAI/finbert · Hugging Face Transformers
Purpose
Classifies tone of earnings call transcripts and MD&A sections as positive / negative / neutral at the sentence level. Aggregated scores feed: Sentiment Alpha (M7), Credibility Alpha (confidence trajectory), and Risk Score (management distress signals).
Additional Signals
Tracks sentiment trajectory over multiple quarters — a deteriorating tone that precedes financial results is an early warning signal. Uncertainty language detection (hedging, qualifications) feeds the Credibility Alpha.
Core NLP RAG-adjacent
Model 11 · Core
Consistency Model
Variance / Coefficient of Variation Analysis
Purpose
Measures stability of growth metrics across years. Computes coefficient of variation (CV = σ/μ) for revenue growth, margin, and ROE. Low CV = consistent performance = positive Consistency Alpha. Erratic CV = risk flag. Specifically separates structural consistency from cyclical patterns.
Core Quant Alpha Feed

Model Group 12–14 · Optional
Advanced Optional Models (V1.5+)

These models are not required for V1.0 and should not block the initial build. They are included in the architecture to define the upgrade path and ensure the codebase is designed to accommodate them without structural refactoring.

Model 12 · Optional · V1.5
Random Forest / XGBoost Classifier
scikit-learn RF · XGBoost · MLflow versioning
Purpose
Learns non-linear patterns in financial features to classify companies into risk buckets or predict earnings beat/miss probability. Requires labelled historical data for training. Supplements but does not replace the rule-based risk scorer — adds ML depth where patterns are non-obvious.
Optional Risk
Model 13 · Optional · V1.5
CAPM / Beta Factor Model
OLS Regression on Market Returns
Purpose
Computes the company's systematic risk (β) relative to a benchmark index (NIFTY 50). Adds institutional-level depth to the risk analysis. Requires market price data — flagged as optional because EREBUS is primarily fundamental, not market-price driven. Add when price data pipeline is established.
Optional Quant
Model 14 · Optional · V2.0
NLI Entailment Verifier
DeBERTa-v3-NLI · Cross-Encoder Entailment
Purpose
Validates that LLM-generated insights are logically entailed by the retrieved source documents. Catches hallucinations where the LLM generates plausible-sounding but unsupported claims. Each generated sentence is scored: ENTAILED / NEUTRAL / CONTRADICTED vs source chunks. Critical for institutional trust.

Scope Boundary — Critical
Models Excluded
Scope Enforcement

The following model categories are explicitly out of scope for EREBUS PS2. Including them would misrepresent the platform as a trading or portfolio management system — which it is not. These exclusions are final and should not be overridden under client pressure to "add alpha."

✕ Portfolio Optimisation Models — Markowitz Mean-Variance, Risk Parity, Black-Litterman ALLOCATION / OUT OF SCOPE
✕ Price Prediction Models — LSTM price forecasting, Prophet, ARIMA on stock prices TRADING / OUT OF SCOPE
✕ Trading Strategy Models — momentum strategies, mean-reversion signals, pairs trading TRADING / OUT OF SCOPE
✕ Reinforcement Learning Trading Systems — RL agents optimising trade execution or P&L TRADING / OUT OF SCOPE
✕ Options Pricing Models — Black-Scholes, Greeks, implied volatility surfaces DERIVATIVES / OUT OF SCOPE
